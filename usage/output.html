Details of FitHiChIP outputs¶

Summary HTML file¶

Suppose OutDir denotes the specified output directory in the configuration file.

Then, successful execution of FitHiChIP pipeline generates an HTML file Summary_results_FitHiChIP.html within this output directory OutDir.

This file lists all output files generated from the FitHiChIP pipeline, according to the given input parameters.

Details of output files¶

Here we describe all the files and folders existing within the specified output directory OutDir.

User may additionally open this page Setting up configuration file for checking the configuration parameters and analyzing the below mentioned output files and folders:

  1. Parameters.txt: file listing the parameters used.

  2. HiCPro_Matrix_BinSize”BINSIZE”:

    If the parameters (A.2) and (A.3) are empty, this folder contains the interaction matrix files generated from the input valid pairs file.

    The interaction matrix files are MatrixHiCPro_abs.bed and MatrixHiCPro.matrix.

    • *.interactions.initial.bed: CIS locus pairs along with their contact counts, corresponding to the input valid pairs (or contact matrix).

    • L_”l”_U”u”: folder name according to the distance thresholds “l” and “u” specified in the parameters (B.7) and (B.8).

      • *.cis.interactions.DistThr.bed: File containing CIS interactions within the distance range from “l” to “u”.

  1. NormFeatures:

    Folder containing various statistical features computed for individual genomic intervals / bins (with respect to the specified bin size).

    • User should specifically look at the file *.AllBin_CompleteFeat.bed, having the following format:

      • Columns 1 to 3: chromosome bin.
      • Column 4: Coverage of this bin (number of reads mapped onto it)
      • Column 5: if 1, this bin is a peak-bin (overlapping with the reference peaks, provided in the parameter B.1); otherwise (0) non-peak bin.
      • Column 6: Bias value of a bin; either coverage or ICE bias depending on the parameter B.11
      • Columns 7 and 8: unused columns.

  2. FitHiChIP_”INTTYPE”_b”BINSIZE”_L”l”_U”u”

    Folder according to the specified foreground (type of interactions to be reported) as per the parameter B.5, bin size (parameter B.6), and the distance thresholds (parameters B.7 and B.8)

    • INTTYPE is a string with values ALL2ALL (when the parameter (B.5) = 4), Peak2Peak (parameter (B.5) = 1), Peak2NonPeak (parameter (B.5) = 2), or

    Peak2ALL (parameter (B.5) = 3).

    • Within this folder, a directory structure of the following name (pattern) is created:

      P2PBckgr_[UseP2PBackgrnd]/[BiasType]

    Where:

    • [UseP2PBackgrnd]: Value of the parameter (B.10)
    • [BiasType]: A string, either “Coverage_Bias” or “ICE_Bias”, according to the parameter (B.11).

    • Within this directory, following files are important:

      • Interactions.sortedGenDist.bed: Interactions sorted by increasing genomic distance. Used as the input to FitHiChIP.
      • FitHiC_BiasCorr:

      Within this folder, following files and folders reside:

      • Bin_Info.log: Equal occupancy bins employed in FitHiChIP

      • configuration.txt: list of configuration parameters for FitHiChIP

      • *PREFIX*.interactions_FitHiC.bed: Lists all interactions along with their significance (p-value and FDR) computed using FitHiChIP.

        Note

        For differential analysis of HiChIP loops (page Differential analysis of HiChIP loops), this file is to be provided as input.

      • *PREFIX*.interactions_FitHiC_Q”qval”.bed:

        Significant interactions according to the q-value threshold (qval) specified in parameter (B.9).

        For example, if qval=0.01 (default), the file name becomes *PREFIX*.interactions_FitHiC_Q0.01.bed.

        Note

        1. This file is the set of significant interactions generated by FitHiChIP, without applying any merge filtering.

        2. For peak-to-all foreground (default output mode), if UseP2PBackgrnd = 0, then this set of loops refer to FitHiChIP(L).

        3. If UseP2PBackgrnd = 1, this set of loops refer to FitHiChIP(S) output.

        4. For a custom FDR threshold (say 0.05), user may use the following awk script to generate the significant interactions.

          awk ‘$NF<0.05’ PREFIX.interactions_FitHiC.bed > FitHiChIP_out_Q0.05.bed

      • *PREFIX*.interactions_FitHiC_Q”qval”_WashU.bed.*:

        Significant FitHiChIP loops converted to WashU epigenome browser compatible format.

      • Merge_Nearby_Interactions:

        Directory containing the output of merge filtering operation, if the parameter (B.12) = 1.

        • *PREFIX*.interactions_FitHiC_Q”qval”_MergeNearContacts.bed:

          FitHiChIP significant loops returned after the merge filtering operation.

          Note

          1. For peak-to-all foreground (default output mode), if UseP2PBackgrnd = 0, this set of loops refer to FitHiChIP(L+M) output.
          2. If UseP2PBackgrnd = 1, this set of loops are denoted as FitHiChIP(S+M) in the manuscript.

        • *PREFIX*.interactions_FitHiC_Q”qval”_MergeNearContacts_WashU.bed.*:

          WashU epigenome browser compatible loops.

Visualizing significant interactions in WashU epigenome browser¶

FitHiChIP interactions can be loaded in the WashU epigenome browser for visualizing the significant interactions.

Note

Two different versions of WashU epigenome browser currently exist:

current (<http://epigenomegateway.wustl.edu/browser/>) and

legacy (<http://epigenomegateway.wustl.edu/legacy/>).

These two versions have substantial differences, so we describe using both of these versions.

Using current version¶

The files *_WashU.bed.gz mentioned above (FitHiChIP generated significant loops, with or without applying merged filtering) are all compatible with the current version. User can visualize these tracks by following these steps:

  1. Open <http://epigenomegateway.wustl.edu/browser/>
  2. Select reference genome.
  3. select Tracks -> upload Local Track
  4. Choose track file type: longrange
  5. Choose track file: select both *_WashU.bed.gz and *_WashU.bed.gz.tbi files together (2 files).
  6. Close the track upload box (right corner - (X) sign)
  7. Now right click on the track name, and select DISPLAY MODE as ARC

Using legacy version¶

User would need to convert the FitHiChIP significant loops (in .bed format, not gzipped version) as the following:

  1. Output of FitHiChIP(L) or FitHiChIP(S) - file FitHiC_Q”qval”.bed

    Let us denote the file as inp.bed. It contains header line, first 6 fields denote the interacting bins and the last field denotes the q-value. User needs to apply the following awk script to this file:

    awk -F[‘t’] ‘{if (NR > 1) {if ($NF > 0) {print $1”,”(($2+$3)/2-1)”,”(($2+$3)/2+1)”t”$4”,”(($5+$6)/2-1)”,”(($5+$6)/2+1)”t”(-log($NF)/log(10))} else {print $1”,”(($2+$3)/2-1)”,”(($2+$3)/2+1)”t”$4”,”(($5+$6)/2-1)”,”(($5+$6)/2+1)”t500”}}}’ inp.bed > out_WashU.bed

  2. Output of FitHiChIP(L+M) or FitHiChIP(S+M) - file FitHiC_Q”qval”_MergeNearContacts.bed

    Let us denote the file as inp_merged.bed. It contains header line, first 6 fields denote the interacting bins and the 9th field stores the q-value. User needs to apply the following awk script to this file:

    awk -F[‘t’] ‘{if (NR > 1) {if ($9 > 0) {print $1”,”(($2+$3)/2-1)”,”(($2+$3)/2+1)”t”$4”,”(($5+$6)/2-1)”,”(($5+$6)/2+1)”t”(-log($9)/log(10))} else {print $1”,”(($2+$3)/2-1)”,”(($2+$3)/2+1)”t”$4”,”(($5+$6)/2-1)”,”(($5+$6)/2+1)”t500”}}}’ inp_merged.bed > out_merged_WashU.bed

Now user needs to upload the generated .bed file (Note: we do not need to convert them in gzipped / tabix formats) in the epigenome browser, by performing the following steps.

  1. Open <http://epigenomegateway.wustl.edu/legacy/>
  2. Select reference genome.
  3. Select Genome Browser -> Apps -> File upload -> Choose files
  4. Select the generated WashU bed file and click setup
  5. Select Radio button for Pairwise interaction -> add as Track
  6. Now right click on the track in the browser, and select Arc.

Visualizing significant interactions in UCSC genome browser¶

FitHiChIP loops can be converted to the bigBed format (or precisely BigInteract format) for visualizing in UCSC genome browser. User should perform the following steps:

  1. Check the link <https://genome.ucsc.edu/goldenPath/help/interact.html>, example 4, for reference.

  2. Download UCSC bedToBigBed utility function, store it in system.

  3. Download the file interact.as (specified in the above mentioned link) and store it in the system.

  4. Then, to convert the FitHiChIP(L) or FitHiChIP(S) generated significant interactions (file FitHiC_Q”qval”.bed) into bed format, using the following command:

    awk -F’[t]’ ‘{if (NR>1) {if ($NF>0) {a=int(-log($NF)/log(10)) + 1} else {a=500}; b=100; print $1”t”(($2+$3)/2-1)”t”(($5+$6)/2+1)”t.t”a”t”b”t.t#0000FFt”$1”t”(($2+$3)/2-1)”t”(($2+$3)/2+1)”t.t.t”$4”t”(($5+$6)/2-1)”t”(($5+$6)/2+1)”t.t.”}}’ FitHiC_Q”qval”.bed | sort -k1,1 -k2,2n -k3,3n > out_FitHiChIP_converted.bed

  5. Then convert the generated file into BigBed format as

    /path/to/bedToBigBed -as=/path/to/interact.as -type=bed5+13 out_FitHiChIP_converted.bed ref_genome_chr.sizes out_loop.bb

    where,

    1. /path/to/bedToBigBed: path of bedToBigBed utility in local system
    2. /path/to/interact.as: path of the file interact.as in local system
    3. ref_genome_chr.sizes: Same as the parameter (B.4) (ChrSizeFile) in FitHiChIP configuration options.
    4. The file out_loop.bb is the BigBed converted significant interactions. User can upload this file in UCSC genome browser for visualization.

6. To convert the output of FitHiChIP(L+M) or FitHiChIP(S+M) (file FitHiC_Q”qval”_MergeNearContacts.bed), replace “$NF” with “$9” in the command mentioned in (4). Rest of the procedure is similar.
